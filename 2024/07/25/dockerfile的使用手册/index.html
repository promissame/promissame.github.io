dockerfile使用方法

在我们使用docker的时候，总是会有人让你使用docker-compose，而不是docker。  
docker-compose其实是docker的升级版，它能同时挂载很多个容器，比docker的能力强。

什么是容器

容器你可以当作是一个虚拟机，但是它比虚拟机消耗的性能更少，更轻量。

docker-compose还是docker compose

在网络上，几乎大部分人都是使用docker-compose而不是docker compose，但是当你在linux系统中安装了最新版docker，输入docker-compose却报错了：  
-bash: docker-compose: command not found
然后你就会开始baidu.com了，现在停下来，不要浪费时间了，最新版的docker将会以插件形式安装docker-compose，在linux系统上使用docker compose即可。但是docker desktop对于基于windows系统来说，docker-compose仍然是软件安装，使用docker-compose命令。

使用docker-compose up

docker-compose up(linux系统使用docker compose up)可以一键加载所有的镜像文件，但是这其实是有个配置文件的，它将提供数据给docker-ompose build、docker-compose down、docker-compose up等命令。

这是一个最基础的docker-compose.yml模板，基本可以对于一个容器，对于初学者来说，只需要使用这些。更多需求。  
在XXXX填入你的容器端口和映射端口，yours处填写你的容器名，./Dockerfile处填写你的dockerfile路径。

dockerfile内容

dockerfile就是你容器的镜像，可以理解为容器需要执行的操作。  
我们通常使用FORM，RUN，COPY，WORKDIR，CMD命令。
其中FORM是你需要使用的镜像文件，如果我们需要使用python，就需要FORM python3.8-slim获取python的镜像，在容器中安装python3.8。  
然后你需要设置你的容器的工作目录，因为docker容器创建的是一个linux虚拟机，你需要建立一个工作文件夹。
WORKDIR /files，在files填写你的工作目录，这就是你容器的工作目录，你在dockerfile执行的操作全部在这个文件夹中执行。  
如果你需要执行一些初始化命令，例如给你的容器安装依赖文件和环境，那么你需要使用RUN命令。

处理错误

在运行docker compose up(docker-compose up)的时候，我们往往会遇到报错，根据本人构建docker容器的经历。  

  1. 检查你的dockerfile，是否有代码错误或者路径错误。
  2. 保证你的docker服务正常运行，windows系统检查你的docker desktop正常运行。如果没有，使用阿里云的加速镜像。
  3. 保证docker的版本正确，使用docker -v检查版本，新版本的docker自动安装docker compose，如果提示docker-compose命令不存在，请使用另外一个docker compos命令。(docker-compose不存在使用docker compose，docker compose不存在使用docker-compose)。
  4. 重启docker服务，往往会解决很多问题。
  5. 关于javascript运行内存不够导致的build失败，我们在下一个部分提出解决方案。
  6. 编写一个循环执行的python文件，让它反复输出文件，这样你就可以进入docker容器内查看文件，查找错误。往往用于文件不存在的情况下。
